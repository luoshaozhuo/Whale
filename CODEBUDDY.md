# Whale - 能源数据统一平台

## 沟通风格约束
- 回答简练，言简意赅
- 不直接贴详细代码（除非明确要求）
- 聚焦核心问题，避免冗长解释

## 开发方法论
- **渐进式开发**：不要一次性实现复杂的系统，小步迭代
- **不过度设计**：按需实现，避免过早做性能优化
- **阶段性重构**：功能稳定后进行架构优化
- **测试驱动**：确保每次迭代可靠

## Python编码规范
- **标准**：PEP 8
- **格式化**：Black（行宽100）
- **检查**：Flake8 + mypy
- **文档**：Google风格文档字符串
- **类型提示**：全面使用Python 3.10+语法

## 项目约定
1. **架构演进**：随功能增加逐步分层
2. **代码质量**：每次提交前通过代码检查
3. **团队协作**：配置纳入版本控制

## 代码质量自动化
- **自动检查**：每次代码生成后，自动运行质量检查
- **Skill 位置**：项目内 `.codebuddy/skills/python-code-quality-checker/`
- **Skill 描述**：使用 Black、Flake8、isort、mypy 检查和修复 Python 代码质量，确保符合 PEP 8 和项目配置标准
- **工作方式**：完全依赖 LLM 能力进行分析、检查和修复
- **检查内容**：
  - Black 格式化（行宽100）
  - isort 导入排序
  - Flake8 代码风格检查
  - mypy 类型检查
- **工作流程**：
  1. CodeBuddy 生成代码
  2. 自动触发项目内质量检查 Skill
  3. LLM 分析代码质量并识别问题
  4. LLM 提供修复方案并直接应用修复
  5. 确保代码完全符合项目标准
