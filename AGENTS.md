# Whale 项目说明（Codex）

先读取仓库根目录的 `CODEBUDDY.md`，并将其作为本项目的主约束与工作方式说明。

在执行任何会修改仓库内容的任务前，必须先读取并遵守本文件中引用的约束文件；不得跳过读取步骤后直接依赖历史记忆、惯性做法或检查器默认行为。

引用规则文件中的文本要求优先于“我以为通常应该怎样做”的经验判断。若规则文本与 lint / formatter / type checker 的实际检查范围不一致，必须以规则文本为准执行，并将工具结果仅作为辅助验证，而不能将“检查通过”等同于“已经满足规则”。

当用户要求“按规则处理”“补注释”“补文档字符串”或类似事项时，必须显式对照对应规则文件逐项落实，不得只做最小 lint 修复后宣称完成。

在编写或修改 Python 代码时，额外读取并遵守以下规则文件：

- `.codebuddy/rules/general.md`
- `.codebuddy/rules/python-style.md`
- `.codebuddy/rules/python-typing.md`
- `.codebuddy/rules/python-docstrings.md`
- `.codebuddy/rules/testing-policy.md`

对上述引用文件的执行要求：

- 必须把这些文件视为 Python 代码修改任务的直接约束，而不是参考资料。
- 若发现引用文件缺失、重命名或内容与仓库现状不一致，必须先向用户说明，再在现有可读规则范围内继续处理，不能静默忽略。
- 修改完成后，必须同时说明“按哪些引用文件执行”与“做了哪些验证”；不能只汇报工具通过情况。
