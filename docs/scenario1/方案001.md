# Whale 场景一最终实施方案（可落地版）

---

# 1. 场景目标

本场景目标：

构建一个**最小但完整的风机数据接入与分层数据链路闭环**，实现：

$$
\text{SCL初始化} \rightarrow \text{OPC UA采集} \rightarrow \text{ODS} \rightarrow \text{标准化} \rightarrow \text{清洗} \rightarrow \text{DWD} \rightarrow \text{DWS} \rightarrow \text{ADS}
$$

并满足：

- 可运行
- 可测试（严格符合 testing-policy）
- 可演进（但不提前过度设计）

---

# 3. 场景边界

## 输入

- 最小 SCL 文件（定义测点）
- OPC UA Server（mock 或真实）

## 输出

- ODS：原始数据
- DWD：清洗后明细
- DWS：实时/定期聚合
- ADS：业务分析

## 不做

- HA / 分布式
- Kafka
- 动态采样率
- 权限/监控
- 大规模性能

---

# 4. 最小 SCL 子集（强制约束）

仅支持以下字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| turbine_id | 是 | 风机标识 |
| point_code | 是 | 测点编码 |
| opcua_node_id | 是 | OPC UA 节点 |
| value_type | 是 | 数据类型 |
| unit | 是 | 单位 |
| min_value | 否 | 下限 |
| max_value | 否 | 上限 |
| deadband | 否 | 死区 |
| max_rate_of_change | 否 | 变化率 |
| aggregate_group | 否 | 聚合分组 |

约束：

- 缺少必填字段 → 抛异常
- 多余字段 → 忽略
- 不支持 IEC61850 复杂结构

| event_time | 源数据时间 |
| ingest_time | 平台接收时间 |

- DWD/DWS/ADS → 使用 event_time
- ODS → 使用 ingest_time
- 场景一不处理乱序（只记录）

# 10. 并发模型

```
Collector
  ↓
RawQueue
  ↓
Pipeline
  ↓
Dispatch
  ├── DWD
  ├── RealtimeAgg
  └── PeriodicAgg
```

约束：

- 不允许多个线程竞争消费同一数据
- 必须统一 dispatch

---

# 11. 清洗规则

顺序执行：

1. 空值校验
2. 范围校验
3. 死区
4. 变化率

输出必须包含：

- quality_code
- clean_reason
